# P11-T06 - Crash/Panic Hygiene

Issue: PER-95 / P11-T06.

## Requirements Summary

Source context:
- PER-95 requires crash/panic hygiene: panic logs must be redacted and must not emit raw store dumps.
- The production master plan path named by the issue is absent in this checkout; the active anchors are the issue body/metadata, `docs/release/handoff-2026-06-10-current-state.md`, `.omc/plans/discrypt-plan.md`, and adjacent Phase 11 plans/evidence for PER-90, PER-93, and PER-94.
- PER-90 centralized structured command-error redaction in `apps/desktop/src-tauri/src/lib.rs`; PER-95 should reuse that boundary rather than introducing independent string handling.

## Acceptance Criteria

- Tauri startup installs a panic hook before app runtime work starts.
- Panic output is a structured `discrypt.panic.v1` JSON entry with timestamp, stable code, redacted message, recovery hint, and redacted low-cardinality context.
- Panic diagnostics omit raw backtraces, raw file paths, raw app-state/store dumps, vault/keyring material, provider credentials, raw SDP/ICE/TURN secrets, OpenMLS secrets/Welcome payloads, plaintext/media material, and raw member identifiers.
- `export_diagnostics_log()` includes the latest redacted panic entry for support bundles.
- Existing unreadable storage behavior remains fail-closed; panic handling does not create replacement storage or claim recovery.

## Implementation Steps

1. Add a serializable `PanicLogEntry` and a process-local latest panic slot in `apps/desktop/src-tauri/src/lib.rs`.
2. Build panic entries through the existing `redact_sensitive_observable_copy` and `redact_command_error_context` helpers.
3. Install the panic hook at the start of Tauri `run()` and emit only the structured redacted line, without chaining the default raw panic hook.
4. Include `structured_logs.last_panic` in `export_diagnostics_log()`.
5. Add a forced-panic regression test that captures a panic with seeded forbidden strings, writes `target/per95-crash-panic-hygiene/panic-diagnostics-log.json`, and scans the structured line plus support bundle for raw forbidden values.
6. Add release evidence under `docs/release/` and verify with targeted Rust/Tauri tests, formatting, diff check, and explicit artifact scan.

## Failure Modes And Safety

- Non-string panic payloads are reported as `non_string_panic_payload` with no raw Debug output.
- Source location is converted to a redacted file reference plus line/column numbers; absolute paths are not emitted.
- Backtraces are intentionally omitted from the structured panic entry because environment variables, paths, and stack frames may contain sensitive local detail.
- Panic handling is diagnostic-only and does not mutate storage, vault/keyring state, OpenMLS state, transport state, or UI truth claims.
