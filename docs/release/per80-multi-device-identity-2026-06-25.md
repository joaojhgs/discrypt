# PER-80 Multi-Device Identity Evidence

Date: 2026-06-25

## Scope

PER-80 / P9-T06 implements local backend harness evidence for multi-device identity. The proof is in `discrypt-mls-core` and covers an existing-device-authorized second device joining the same account as its own OpenMLS leaf, backend transparency events, durable reload, and remove/rekey denial.

This is local Rust/OpenMLS harness evidence. It is not production split-machine evidence, a UI device-management implementation, backup/recovery, adapter behavior, packaging, or release readiness.

## Evidence Added

- `OpenMlsGroupEngine::member_labels` exposes backend-observed OpenMLS credential labels so callers/tests can distinguish real group membership from frontend-only labels.
- `two_device_profile_pairing_join_reload_and_remove_rekey` composes:
  - `DeviceSet` existing-device pairing signed by the account identity.
  - `alice:phone` generated by the joining device provider and admitted with a real OpenMLS Welcome.
  - separate OpenMLS labels for `alice:laptop` and `alice:phone`.
  - serialized/reloaded device-set state and SQLite-backed OpenMLS group reload.
  - backend transparency events for device add/pair/remove.
  - OpenMLS remove commit, epoch/exporter rotation, and fail-closed post-removal commit application on the removed device.

## Retention/Shred Boundary

PER-78 retention and PER-79 crypto-shred semantics are not changed. Cross-device shred remains cooperative and best-effort for honest own devices: online own devices can converge immediately, while offline own devices remain pending until they reconnect and sync tombstones. No evidence in this task claims deleted everywhere or resurrects shredded/locked content.
